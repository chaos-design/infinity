# 安全策略

## 支持范围

Infinity 仍处于早期版本阶段。安全报告默认面向当前 `main` 分支和最新 GitHub Release。

| 版本 | 支持状态 |
| --- | --- |
| 最新 Release | 支持 |
| `main` 分支 | 支持 |
| 旧版本 | 尽力处理 |

## 报告漏洞

如果你发现疑似安全问题，请不要创建公开 Issue。请优先使用 GitHub Security Advisory 私密报告:

https://github.com/chaos-design/infinity/security/advisories/new

报告时请尽量提供:

- 漏洞摘要和影响范围。
- 复现步骤或最小复现示例。
- 受影响的浏览器、系统、扩展版本或 commit。
- 可能的修复建议或缓解方式。

## 处理流程

1. 维护者会确认报告是否可复现，并评估影响范围。
2. 如确认为安全问题，会在私密渠道讨论修复方案。
3. 修复完成后，会通过 Release、提交记录或安全公告说明变更。
4. 在修复公开前，请避免披露漏洞细节。

## 安全边界

Infinity 是浏览器新标签页扩展，主要使用 `chrome.storage.local` 保存设置、快捷方式和标签关系。请特别关注以下风险:

- 不可信 URL 或标题在 UI 中展示时的转义问题。
- 浏览器权限变更带来的数据访问范围扩大。
- 扩展打包、Release 资源或依赖供应链风险。
- 用户本地数据的误删、泄露或跨域暴露风险。
